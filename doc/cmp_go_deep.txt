*cmp_go_deep-options*

DESCRIPTION
    This section describes the default options for the `cmp_go_deep` source that provides symbol
    completions for Go, integrating with LSP (gopls) and Tree-sitter.

DEFAULT OPTIONS
    The following options can be configured:

    - `get_documentation_implementation`        *string*  
        Whether to get documentation with LSP `textDocument/hover`, or extract it with regex.  
        Default: `"hover"`  
        Options: `"hover"`, `"regex"`

    - `documentation_wait_timeout_ms`           *integer*  
        Timeout in milliseconds for getting documentation.  
        Default: `500`  
        Note: Only used when `get_documentation_implementation` is not `"regex"`.

    - `add_import_statement_implementation`     *string*  
        Whether to add import statement with `gopls.add_import`, or manually with the help of Tree-sitter.  
        Default: `"treesitter"`  
        Options: `"gopls"`, `"treesitter"`  
        Note: The `gopls.add_import` command sometimes noops.

    - `get_package_name_implementation`         *string*  
        Whether to get the package name with `treesitter` or `regex`.  
        Default: `"regex"`  
        Options: `"treesitter"`, `"regex"`
        Note: The `treesitter` implementation is more accurate, but slower.
        Known issues: The `regex` implementation doesn't work for package names declared like: `/* hehe */ package xd`

    - `exclude_vendored_packages`               *boolean*  
        Whether to exclude vendored packages from completions.  
        Default: `false`  
        Note: Enabling this option has a negligible performance impact.
